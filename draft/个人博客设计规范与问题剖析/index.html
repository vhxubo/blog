如何修改时区

时区是一个很严重的问题, 在 GitHub 上主要使用的 UTC 时间, 所以在生成页面的时候也是 UTC 时间, 甚至说, Liquid 里面的 data filter 似乎也是格式化的 UTC 时间, 那这样对于读者的观感就会出现偏差, "这创建和更新时间也太不符合逻辑了吧"

eleventy 里面没有提供自定义 timezone 的方案, 但是我们可以基于 dayjs 设计一个新的 filter 供格式化时间使用

// .eleventy.js 
const dayjs = require("dayjs");
const utc = require("dayjs/plugin/utc");
const timezone = require("dayjs/plugin/timezone");
dayjs.extend(utc);
dayjs.extend(timezone);

module.exports = function (eleventyConfig) {
  eleventyConfig.addFilter("datetzday", function (date) {
    return dayjs(date).tz("Asia/Shanghai").format("YYYY-MM-DD");
  });
  eleventyConfig.addFilter("datetztime", function (date) {
    return dayjs(date).tz("Asia/Shanghai").format("YYYY-MM-DD HH:mm");
  });
};

字体优化

本博客使用了鸿蒙字体, HarmonyOS_Sans_SC_Regular, 从官网下载的字体是 ttf 格式, 体积比较大, 可以通过转换为 woff woff2, 有所体积, 并通过链接使用, 并配置预先加载, 缩短字体替换时间

通过预加载 web 字体提高加载速度

需要注意的是, 需要配置相同的链接, 在 preload 和 @font-face 中

  <link
    rel="preload"
    href="https://cdn.jsdelivr.net/gh/vhxubo/blog@main/fonts/HarmonyOS_Sans_SC_Regular.woff2"
    as="font"
    type="font/woff2"
    crossorigin
  />
  <link
    rel="preload"
    href="https://cdn.jsdelivr.net/gh/vhxubo/blog@main/fonts/HarmonyOS_Sans_SC_Regular.woff"
    as="font"
    type="font/woff"
    crossorigin
  />
@font-face {
  font-family: HarmonyOS_Sans_SC_Regular;
  font-style: normal;
  font-display: swap; /* 不影响正常字体显示, 新字体加载完成后, 自动替换 */
  src: url("https://cdn.jsdelivr.net/gh/vhxubo/blog@main/fonts/HarmonyOS_Sans_SC_Regular.woff2")
      format("woff2"),
    url("https://cdn.jsdelivr.net/gh/vhxubo/blog@main/fonts/HarmonyOS_Sans_SC_Regular.woff")
      format("woff");
}
